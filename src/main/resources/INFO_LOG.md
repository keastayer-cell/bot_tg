# Telegram Bot - @tg_tanya_bot

## Описание

Бот для автоматической рассылки сообщений и картинок подписчикам в заданное время.

## Возможности

- **Авторассылка** — бот автоматически отправляет контент подписчикам в указанное время (по умолчанию 09:00)
- **Управление подписчиками** — админ может добавлять/удалять получателей рассылки
- **Очередь сообщений** — тексты и картинки добавляются в очередь и отправляются по очереди
- **Обратная связь** — подписчики могут отправлять сообщения админу
- **Статистика** — просмотр количества сообщений, картинок и получателей

## 📋 Примеры использования

### Пример 1: Добавление картинки и отправка

```
1. Админ отправляет фото боту → фото сохраняется в БД
2. Админ пишет /images → видит список картинок
3. Админ пишет /sendimage 1 → картинка отправляется всем подписчикам и удаляется
4. Админ пишет /images → картинка больше не в списке
```

### Пример 2: Заполнение очереди и рассылка

```
1. Админ добавляет тексты: /add Утро!, /add Привет!
2. Админ добавляет картинки: отправляет 2 фото
3. Админ пишет /fillqueue → очередь заполняется смешанным контентом
4. Админ пишет /stats → видит 2 текста, 2 картинки, 4 элемента в очереди
5. Бот в 09:00 берёт первый элемент и отправляет всем
6. Элемент автоматически удаляется из очереди
```

### Пример 3: Срочная отправка

```
1. Админ пишет /queue → видит текущую очередь
2. Админ пишет /now → первый элемент отправляется НЕМЕДЛЕННО
3. Элемент удаляется из очереди
4. /now отправляет следующий элемент
```

## Как это работает

### 🔄 Основной цикл

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1️⃣  Админ добавляет контент (/add текст или фото)    │
│      ↓                                                  │
│  2️⃣  Контент добавляется в ОЧЕРЕДЬ                     │
│      ↓                                                  │
│  3️⃣  Каждую минуту бот проверяет время                │
│      ↓                                                  │
│  4️⃣  В нужное время (/time) бот берет элемент         │
│      ↓                                                  │
│  5️⃣  Отправляет ВСЕ подписчикам                      │
│      ↓                                                  │
│  6️⃣  Элемент удаляется из очереди                      │
│      ↓                                                  │
│  7️⃣  Повтор: идет на пункт 4️⃣                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 📌 Типы контента в очереди

```
Очередь может содержать:

📝 Текстовые сообщения
   └─ добавляются командой /add
   └─ один элемент = одно сообщение

🖼️  Картинки (фото)
   └─ добавляются отправкой фото боту
   └─ один элемент = одно фото
   └─ АВТОМАТИЧЕСКИ УДАЛЯЕТСЯ после отправки
   └─ Нельзя отправить два раза!

⚠️  Текущее ограничение: нельзя удалить элемент из очереди вручную
   └─ Только полная рассылка через /now
```

### 💾 Хранилище данных

```
PostgreSQL (на Railway):
  ├─ recipients            ← список подписчиков (chat ID)
  ├─ message_queue         ← очередь текстов и картинок
  ├─ message              ← сохранённые тексты
  └─ image_index          ← индекс картинок (Telegram file_id)

Файловая система:
  └─ application.conf     ← конфиг (время, админ ID, timezone)
```

### 🔐 Права доступа

```
АДМИН (по ADMIN_CHAT_ID):
  ✅ Все команды
  ✅ Добавление/удаление подписчиков
  ✅ Управление очередью
  ✅ Отправка контента

ПОДПИСЧИК (добавлен админом):
  ✅ Получает рассылку в нужное время
  ✅ Может написать админу (кнопка "📬 Админу")
  ❌ Больше ничего не может

НОВЫЙ ПОЛЬЗОВАТЕЛЬ:
  ✅ Может нажать /start для подписки
  ❌ После подписки = обычный подписчик
```

### ⏰ Планирование отправки

```
Примеры:

/settime 09:00       ← отправка в 09:00 каждый день
/settime 14:30       ← отправка в 14:30 каждый день
/time                ← посмотреть текущее время

Механизм:
  1. Каждую минуту MessageScheduler проверяет текущее время
  2. Если текущее время = установленное время
  3. И очередь не пуста
  4. И ещё не был отправлен в эту минуту (дедупликация)
  5. ↓ То берём первый элемент и отправляем
  6. Отправленный элемент удаляется из очереди
```

## 👁️ Мониторинг подписчиков

Бот может отслеживать активность подписчиков и уведомлять админа о их действиях.

### Возможности мониторинга

- **🟢 Онлайн статус** — когда подписчик становится онлайн
- **✍️ Действия** — когда подписчик печатает, записывает голосовое, отправляет фото/видео
- **📊 Статистика** — просмотр активности всех подписчиков
- **🔔 Уведомления** — мгновенные сообщения админу в личку

### Управление мониторингом

```
/monitor                    показать текущий статус и статистику
/monitor on                 включить мониторинг активности
/monitor off                выключить мониторинг активности
/monitor clear              очистить историю активности
```

### Примеры уведомлений

```
🟢 @username (123456789) - онлайн [14:30:25]
✍️ @username (123456789) - печатает [14:30:30]
🎤 @username (123456789) - записывает голосовое [14:31:05]
📷 @username (123456789) - отправляет фото [14:31:15]
```

### Примечания

- Мониторинг работает только для подписчиков (не для админа)
- Уведомления приходят только админу в личные сообщения
- История активности сохраняется в памяти бота
- Можно очистить историю командой `/monitor clear`

# 🏗️ Архитектура сервиса

## 🧩 Основные компоненты

### Telegram Integration Layer
```
MyTelegramBot (extends TelegramLongPollingBot)
  ↓
  Получает обновления от Telegram API
  ↓
  Распределяет на обработчики:
    ├─ handleAdminPhoto()      ← для фото от админа
    ├─ handleAdmin()           ← для команд админа
    ├─ handleRecipient()       ← для сообщений от подписчиков
    └─ sendNow()              ← срочная отправка
  ↓
  Отправляет ответы обратно
```

### Service Layer
```
🔧 Основные сервисы:

ConfigService
  ├─ Управление параметрами (time, timezone, admin-id)
  ├─ Управление подписчиками
  └─ Персистентность конфига в application.conf

MessageService
  ├─ Загрузка/сохранение текстовых сообщений
  └─ Работа с БД таблицей `messages`

ImageService
  ├─ Сохранение Telegram file_id
  ├─ Удаление картинок после отправки
  └─ Работа с БД таблицей `image_index`

QueueService
  ├─ Управление очередью (FIFO)
  ├─ Добавление текстов и картинок
  ├─ Извлечение первого элемента
  └─ Работа с БД таблицей `message_queue`

MessageScheduler
  ├─ Проверка времени каждую минуту
  ├─ Автоматическая отправка в нужное время
  └─ Дедупликация (не отправляет два раза в минуту)
```

### REST Controllers
```
📡 HTTP API endpoints:

PingController (/ping)
  └─ Проверка что сервис живой

InfoController (/info, /info.html)
  ├─ Raw Markdown версия README
  └─ Красиво отрендеренная HTML версия

WebhookController (/webhook)
  └─ Зарезервирован для будущего webhook'а
```

### Database Layer
```
🗄️ PostgreSQL (на Railway) или H2 (fallback)

Таблицы:
  ├─ recipients              UUID → chat ID получателей
  ├─ messages               UUID → сохранённые текстовые сообщения
  ├─ image_index            UUID → Telegram file_id картинок
  ├─ message_queue          UUID → очередь значений (TEXT или IMAGE)
  └─ (автоматически создаются JPA/Hibernate)

### Сценарий 1: Админ добавляет текст

```

        ↓
handleAdmin("/add Привет!")
        ↓
QueueService.addText("Привет!")
        ↓
MessageService.saveMessage("Привет!")
        ↓
QueueItemEntity (TEXT, "Привет!")
        ↓
INSERT в postgresql message_queue
        ↓
Ответ: "✓ Добавлено в очередь"
```

### Сценарий 2: Админ отправляет фото

```
Админ отправляет фото боту
        ↓
MyTelegramBot.onUpdateReceived()
        ↓
handleAdminPhoto(message)
        ↓
ImageService.saveImage(fileId)
        ↓
ImageIndex (file_id = "Ag...")
        ↓
INSERT в postgresql image_index
        ↓
QueueService.addImage(fileId)
        ↓
QueueItemEntity (IMAGE, "Ag...")
        ↓
INSERT в postgresql message_queue
        ↓
Ответ: "✓ Картинка сохранена"
```

### Сценарий 3: Автоматическая отправка в нужное время

```
09:00:00 ← MessageScheduler.check() каждую минуту
        ↓
        ✓ Текущее время = установленное время?
        ✓ Очередь не пуста?
        ✓ Уже отправляли в эту минуту?
        ↓
        ДА на все вопросы!
        ↓
QueueService.popFirst()
        ↓
SELECT первый элемент из message_queue
        ↓
Если IMAGE → ImageService.deleteImage(fileId)
        ↓
DELETE из message_queue
        ↓
ConfigService.setLastSent("2026-04-05 09:00")
        ↓
MyTelegramBot.sendMessage() или sendPhoto()
        └─ ДЛЯ КАЖДОГО подписчика
        ↓
Логирование: "Автоотправка: картинку"
```

### Сценарий 4: Админ отправляет сейчас (/now)

```
Админ пишет: /now
        ↓
MyTelegramBot.handleAdmin("/now")
        ↓
sendNow(chatId)
        ↓
QueueService.popFirst()
        ↓
SELECT первый элемент
        ↓
Если TEXT → отправить текст
Если IMAGE → отправить фото и DELETE из image_index
        ↓
DELETE из message_queue
        ↓
Отправить всем подписчикам
        ↓
Ответ: "✓ Отправлено"
```

   BOT_USERNAME=mybot
   ADMIN_CHAT_ID=987654321

# FAQ

## Q: Как избежать конфликта "getUpdates: [409] Конфликт"?

**A:** Убедись, что бот запущен только в одном месте. Если переходишь между локальным и Railway:
- Останови локальный бот
- Дождись завершения Railway деплоя
- Или удали webhook: `https://api.telegram.org/bot<TOKEN>/deleteWebhook`

## Q: Фото не сохраняются

**A:** Проверь:
- БД доступна: `/stats` показывает >0 картинок?
- PostgreSQL запущен и доступен по `DATABASE_URL`
- Логи: ищи "Картинка сохранена" в Railway логах

## Q: Не работает автоотправка

**A:**
- Проверь время: `/time`
- Убедись, есть получатели: `/recipients`
- Есть что отправлять: `/queue` или `/stats`
- Логи: ищи "checkAndSend" в Railway логах

## Q: Как добавить получателей?

**A:** Только админ может добавлять получателей:
- `/addrecipient <chat_id>` — по ID
- Получатель узнаёт свой ID командой `/chatid`

## Q: Получатель не может выполнить /start

**A:** Это нормально, если получатель уже добавлен админом. Получатель может:
- Читать рассылку
- Написать админу кнопкой "📬 Админу"
- Выполнить `/chatid` для узнавания своего ID

## Q: Как очистить очередь?

**A:** Текущий функционал не поддерживает удаление отдельных элементов. Можно:
- Отправить всё сразу `/now` много раз
- Или удалить строки из БД напрямую

## Q: Можно ли использовать webhook вместо polling?

**A:** Текущая версия использует polling. Поддержка webhook может быть добавлена в будущем релизе.
